# Laravel GUI Parity Matrix

> **Current-state inventory, not target authority:** This matrix records implemented/legacy parity at the hardening-plan baseline. ADR-0017, ADR-0018, ADR-0019 and [`implementation-plan-security-architecture-hardening.md`](implementation-plan-security-architecture-hardening.md) override rows that describe OCR write-back, Chat/RAG, selectable setup origins, optional webhook authentication, raw diagnostics or parallel backends. Do not extend an unsafe `done` row; follow its hardening milestone.

## Follow-up checklist

- Treat every `partial` row as a candidate for a focused follow-up issue before declaring complete GUI parity.
- Prefer Laravel UI/routes as the product surface; keep Python CLI paths for debug/maintenance only.
- When replacing old views, target the durable event-driven model: `commands`, `pipeline_runs`, `pipeline_events`, `pipeline_items`, `actor_executions`, Laravel queues, PostgreSQL and pgvector.
